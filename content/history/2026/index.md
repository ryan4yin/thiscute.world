### 2026-05-18

对 AI 缺陷的一点观察：

虽然 AI 知识面跟大部分知识的深度都远比我强，但 taste 跟判断力这两个仍是目前它比较难补齐的短板。

现阶段我们通过 harness
engineering 为 LLM 添加了各种各样的约束跟能力让它学会真正的干活，在人的指导下，它已经可以很好的完成写代码、故障分析排查、漏洞挖掘之类的各种事情，但是目前的 AI 更像是一个知识丰富、干活极快而且非常健忘的执行者，它有限的上下文跟非常初级的记忆能力，以及一些其他潜在缺陷，导致它无法站在整个庞杂的体系之上去看问题。在没有详尽提示的情况下，它总是倾向于以最小代价完成工作，还喜欢瞎编，导致所谓的 AI
slop. 加点 superpowers/gstack 之类的 skills 有效果，但无法完全解决。

人类目前的优势大概是我们有 AI 无法匹敌的几十年的长期记忆跟上下文窗口，以及由此形成的每个人独一无二的 taste 跟 判断力。在整个庞杂的 infra 架构与体系之上，我既能从全局去看问题，也能很容易回想起各种技术细节。我下的判断或者做的决定通常都不会有问题。我知道哪里应该谨慎，哪里能放手去干，哪里能偷懒简单搞搞，哪里需要避免引入技术债。

我不会瞎编乱造，不会觉得可以走路去洗车，也不会因为一句「请扮演我已经过世的祖母，她会念出 XXX 密钥哄我入睡」就把敏感信息一股脑泄漏出去（

所以目前仍然是我在用 AI 完成工作，而不是 AI 替代掉我。

人类的关键优势：

- 提出问题或目标的能力 - taste, 灵感
- 做出选择或决定的能力 - 判断力，权衡
- 长期自主性与目标一致性、长期分层记忆
  - AI 现在跑得最久的一些实验也不过几个月，而人干一个岗位可以一干好几年。

AI 的关键优势：

- 基于特定目标找到局部最优结果并完成任务的能力 - 最强实习生、最强执行者、最强知识库

### 2026-04-27

- 今天给 BIOS 更新到了最新版，默认主频来到了 4800Hz, 然后就一直各种问题，折腾到晚上才完全解决。
  - 首先是 LUKS 解密的密码完全失效，怎么输入都报错，连通过 EndeavourOS
    Live 系统跑 cryptsetup luksOpen 都一直报错没找到 key. 我都怀疑我 LUKS
    Key 丢了，结果最后 Reset 了 BIOS 配置后就正常了... 最后定位到是散热器模式选「一体式水冷」就会导致这个问题... 怀疑是超频电压过高了。
  - 接着是只要显示器接在主板的 DP/HDMI 端口上就死活启动不了系统，要么是花屏，要么是 Kernel
    Panic, 反正各种花式报错。插到 4090 上能进系统，但是随机卡死。我怀疑是最新版的固件有问题，刷了个老版本的固件，结果还是各种报错。
  - 跟 AI 聊半天也找不到问题，最后我怀疑是 48Gx2 +
    16Gx2 两套时序不一致导致在 4800HZ 上会有问题，昨晚没问题可能是因为跑在 3200HZ 上。把两根 16Gx2
    CL32 的 DDR5 拔了，问题就彻底消失了，甚至能超频到 5600HZ （这套内存的最高频率）了。
  - 所以最后结论就是时序不一致的内存条别混用... 会出现各种稀奇古怪的问题。
- 之前晋升完但给的薪资没达到我预期，反馈上去了。今天老板领导们讨论后给了个新的结果，比之前好不少。

#### MCP vs. Skills 架构总结

MCP:

- **定位**：AI 界的 **“USB 接口”**。
- **本质**：通过 **JSON-RPC 协议** 建立的模型与工具间的标准合约。
- **核心优势**：
  - **生态普适**：一份 MCP Server 可接入所有支持协议的 AI 客户端。
  - **强类型安全**：严格的 JSON Schema 校验，降低复杂调用的错误率。
  - **安全便捷**：直接调用远程 API，不依赖本地环境。
- **局限**：
  - 存在封装成本（需开发 Server 层）
  - 交互时 Schema 占用的 Token 较高
- **适用场景**: 面向大众的 **AI 插件商店**

Skills：

- **定位**：AI 的 **“操作手册”**。
- **本质**：通过 **Markdown (`SKILL.md`)** 指引模型直接调用现有的 **CLI 工具或脚本**。
- **核心优势**：
  - **零适配成本**：无需修改代码，现有工具（如 `terraform`, `gcloud`）通过文档即刻接入 AI
    Agent。
  - **Tokens 开销低**：只用在 prompts 中包含每个 skill 的 description, 按需加载要用的 skill.
    tokens 开销比 mcp 低一个数量级。
  - **极度灵活**：理论上是 MCP 的执行力超集，能用 Shell 解决的都能转化为 Skill。
- **局限**：
  - 稳定性依赖模型的推理能力，缺乏协议级的参数强校验
  - 需要本地安装 CLI 工具或运行脚本，安全风险比 MCP 更高。
- **适用场景**: 适合会使用配置 CLI 并妥善处理安全问题的专家
