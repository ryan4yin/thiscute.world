MCP vs. Skills 架构总结

MCP:

  • 定位：AI 界的 “USB 接口”。
  • 本质：通过 JSON-RPC 协议 建立的模型与工具间的标准合约。
  • 核心优势：
    • 生态普适：一份 MCP Server 可接入所有支持协议的 AI 客户端。
    • 强类型安全：严格的 JSON Schema 校验，降低复杂调用的错误率。
    • 安全便捷：直接调用远程 API，不依赖本地环境。
  • 局限：
    • 存在封装成本（需开发 Server 层）
    • 交互时 Schema 占用的 Token 较高
  • 适用场景: 面向大众的 AI 插件商店

Skills：

  • 定位：AI 的 “操作手册”。
  • 本质：通过 Markdown (SKILL.md) 指引模型直接调用现有的 CLI 工具或脚本。
  • 核心优势：
    • 零适配成本：无需修改代码，现有工具（如 terraform, gcloud）通过文档即刻接入 AI Agent。
    • Tokens 开销低：只用在 prompts 中包含每个 skill 的 description, 按需加载要用的 skill. tokens 开销比 mcp 低一个数量级。
    • 极度灵活：理论上是 MCP 的执行力超集，能用 Shell 解决的都能转化为 Skill。
  • 局限：
    • 稳定性依赖模型的推理能力，缺乏协议级的参数强校验
    • 需要本地安装 CLI 工具或运行脚本，安全风险比 MCP 更高。
  • 适用场景: 适合会使用配置 CLI 并妥善处理安全问题的专家
